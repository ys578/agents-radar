# OpenClaw 项目动态日报 | 2026-06-13

## 1. 今日速览

今日 OpenClaw 项目社区活跃度极高，共产生近千条 Issue 和 PR 更新，但其中大部分为存量问题的持续讨论。**核心动态**在于同时发布了两个新增安全边界补丁的版本 (v2026.6.6 及 beta 版)，表明项目正集中精力解决近期暴露的、涉及多个模块的安全加固问题。然而，**P0 级内存泄漏 (Issue #91588) 和多起 P1 级安全性、数据丢失及关键功能回归问题仍在悬而未决**，社区情绪在功能推进与稳定性焦虑之间摇摆。整体来看，项目处于高强度的迭代期，功能推进迅速，但稳定性与安全性仍是当前主要矛盾。

## 2. 版本发布

今日发布了两个针对安全问题的修补版本：

*   **v2026.6.6 (稳定版) & v2026.6.6-beta.2 (测试版)**
    *   **更新内容**: 两个版本的 Highlights 完全一致，核心是修复一系列安全边界问题。涉及的具体模块包括：`transcripts`（对话记录）、`sandbox binds`（沙箱绑定）、`host environment inheritance`（宿主机环境变量继承）、`MCP stdio`、`Codex HTTP access`、`native search policy`（原生搜索策略）、`elevated sender checks`（发送者权限提升检查）、`deleted-agent ACP bypasses`（已删除代理的访问控制旁路）、`loopback tools`（回环工具）、`Discord moderation`（Discord 审核）以及 `Teams group actions`（Teams 群组操作）等。
    *   **破坏性变更**: 日志中未明确提及，但由于涉及多项安全边界收紧，用户需要注意：
        *   使用 **MCP stdio** 和 **Codex HTTP** 通道的开发者，可能需要调整配置或令牌。
        *   **Discord** 和 **Teams** 等渠道的管理员需检查审核与群组操作的自动化流程是否受影响。
        *   依赖宿主机环境变量（如 `skills.entries.*.env`）的功能可能受到影响，应验证其正常运行。
    *   **迁移注意**: 建议所有用户尽快升级至 `v2026.6.6` 稳定版。升级前，请务必在测试环境验证关键工作流，特别是涉及上述模块的脚本和集成。

## 3. 项目进展

今日合并或关闭了多个对项目有显著推进作用的 PR：

*   **功能推进**:
    *   **多代理会话管理**: PR #75662 将暂停主会话 (`main-session`) 的功能从子代理扩展到了顶层会话，当顶级任务使用 `sessions_yield` 时，主会话能正确进入暂停状态，对复杂工作流编排至关重要。
    *   **安全架构**: PR #92086 引入了“安全矩阵运行时-事实审核模型”，将影响源、能力审批等作为独立输入，为安全策略的强制执行和审计奠定了基础。
    *   **新提供商支持**: PR #92556 增加了 Inworld 作为内置 LLM 提供商，除了现有的 TTS 能力外，还能支持聊天补全、工具调用等。
    *   **消息钩子系统**: 多个 PR（如 #20418、#19922）推进了钩子事件系统，新增了 `session:pre-spawn`、`agent:pre-run`、`message:received` 等事件，允许插件在子代理创建前或消息发送/接收时介入。

## 4. 社区热点

本周最受社区关注的议题集中在**安全性**与**会话状态管理**两大核心痛点：

*   **安全性 (Diamond Lobster 级别)**:

#### 3. OpenClaw 在生态中的定位

*   **优势与定位**: OpenClaw 是当前生态中 **功能最全面、集成最广泛、门槛相对较低** 的旗舰项目。其内置的 Discord/Teams/Web 渠道、沙箱、技能系统等使其成为个人开发者和中小团队的首选“通用型框架”。
*   **技术路线差异**: 与其他项目相比，OpenClaw 更强调 **“平台化”** 和 **“开箱即用”**，通过丰富的“技能”和“渠道”插件来快速落地，而非像 IronClaw/ZeroClaw 那样追求底层的模块化与研发级架构。它在 **“应用层抽象”** 上做得最好。
*   **社区规模与现状**: OpenClaw 拥有所有项目中最庞大的社区活跃度（日Issue/PR近千）。但也正因体量巨大，**稳定性问题（回归、内存泄漏）和社区焦虑情绪也最显著**。其健康度呈现 **“繁荣与风险并存”** 的典型特征。
